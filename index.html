Règle de codification des fichiers contractuels

Référentiel de conformité stricte BIM - Projet ADCRM

PROJET_ PHA_ EMETT_ SPE_ NIV_ TYP_ IND
PROJET
[5 caractères] Code projet défini par le Maître d'Ouvrage.
Exemple : ADCRM
PHA
[3 caractères] Code Phase de projet.
Exemple : PRO
EMETT
[4 caractères] Code de l'émetteur du document.
Exemple : MOEG
SPE
[3 caractères] Code de spécialité.
Exemple : ARC
NIV
[5 caractères] Code de niveau / localisation (chiffres, lettres et tirets de 5 caractères).
Exemple : 00---
TYP
[7 caractères] Code type de document.
Composé de 3 caractères (Alpha) et 4 caractères (Numérique).
Exemple : PLN0001
IND
[1 à 3 caractères] Code indice de document.
Valeurs standards acceptées (ex: -0, -A, AA, 01...)
Note : Convention des Fiches techniques
Pour les Fiches techniques, la convention de nommage spécifique est la suivante :
ADCRM_ EMETT_ TYP_ Numéro de Position_ Description Courte_ IND
